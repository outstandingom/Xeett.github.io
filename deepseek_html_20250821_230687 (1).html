1. Introduction

1.1 Purpose

This document outlines the requirements for the AdKaro Advertising website, a platform for showcasing print services, displaying pricing, and enabling customer inquiries via WhatsApp and email. The website will serve as a digital storefront for the agency's printing services.

1.2 Scope

The website will include:

  • Service catalog (business cards, posters, banners, etc.)
  • Detailed pricing information for all services
  • Contact forms with WhatsApp and email integration
  • Portfolio/gallery of previous work
  • Responsive design for all device types
  • Admin panel for content management

2. Overall Description

2.1 Product Perspective

The AdKaro Advertising website will be a professional platform where potential customers can:

  • Browse available print services and pricing
  • View samples of previous work in a portfolio
  • Contact the agency via multiple channels (form, WhatsApp, email)
  • Request quotes for specific projects

2.2 User Classes

User Type Description
Potential Customers Browse services, view pricing, contact for quotes
Existing Customers Place repeat orders, check status of current orders
Admin/Staff Manage content, view inquiries, update pricing

3. System Features & Requirements

3.1 Functional Requirements

FR1: Service Catalog

FR1.1 Display all print services with categories:
  • Business Cards
  • Posters & Flyers
  • Banners & Signage
  • Brochures & Booklets
  • Merchandise Printing
  • Packaging
FR1.2 Include high-quality images for each service category

FR2: Pricing Display

FR2.1 Clear pricing tables for each service category
FR2.2 Option to show pricing based on quantity tiers
FR2.3 Admin ability to update pricing without developer assistance

FR3: Contact System

FR3.1 Contact form with fields for:
  • Name, Email, Phone
  • Service interested in
  • Message/Project details
FR3.2 Direct WhatsApp integration with pre-filled messages
FR3.3 Email notifications to staff when inquiries received

FR4: Portfolio/Gallery

FR4.1 Showcase previous work with categories
FR4.2 Image gallery with lightbox functionality
FR4.3 Admin ability to add new portfolio items

FR5: Content Management

FR5.1 Admin panel for managing:
  • Service descriptions
  • Pricing information
  • Portfolio items
  • Contact information

3.2 Non-Functional Requirements

Requirement Description
Performance <3s page load time, optimized images
Responsiveness Mobile-first design, works on all screen sizes
Security Form validation, protection against spam
Maintainability Well-documented code, easy content updates
SEO Search engine optimized structure and content
